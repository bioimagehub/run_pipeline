(requires: JOBS Viewer)

(requires: JOBS Viewer)(requires: Well Plate Loader)

Figure 1895. 

Batch Job run provides the ability to queue different jobs for different well plates in plate hotels.

HCA/JOBS > Well Plate > Align Well Plate Holder

(requires: JOBS Viewer)

This function opens the Align Well Plate window. Select a standardized well plate from the database, define the Position of the Locking Clamp to set the well plate orientation and click Next. Then precisely align the well plate by one of the available methods (please see: Well Plates > Align Well Plate Holder). This alignment is used globally in NIS-Elements once the selected well plate is aligned and is recommended for fixed well plate holders.

HCA/JOBS > Well Plate > Manage Custom Well Plates

(requires: JOBS Viewer)

Here you can search for previously saved well plates, create new ones, delete them or import / export them from / to a .BIN file. For more information about how to create custom well plates see Well Plates > Define Plate.

HCA/JOBS > Slide Holder > Align Slide Holder

(requires: JOBS Viewer)

This function opens the Align Slide definition window.

Figure 1896. 

Search for your slide and click Next. Now you have to specify its orientation on the stage. The orientation can be defined by Label Area Position, Smear Direction or Crosses Position (if the slide has crosses). Select the real Orientation of the slide from the combo box. Finally Add two XY points of the two opposite corners of your slide and click Finish.

Slides with covered corners 
If your slide is mounted to the stage in a manner that its corner(s) are not visible in the Live View, it is impossible to align it using the predefined slides from the database. In this scenario there are two options:

  • Create a custom slide (using Sample Holder > Define Slide) with Alignment Crosses and use these crosses to target corners of the coverslip, visible corners of the slide or any other reference points.

  • Use Sample Holder > Define Stage Area with a rectangular shape and find corners of the coverslip, visible corners of the slide or any other visible reference points.

HCA/JOBS > Slide Holder > Manage Custom Slides

(requires: JOBS Viewer)

Here you can search for previously saved slides, create new ones, delete them or import / export them from / to a .BIN file. For more information about how to create custom slides see Sample Holder > Define Slide.

HCA/JOBS > Background Detection > Teach Background/Specimen Detection

(requires: JOBS Viewer)

Opens the Background detection dialog window.

Figure 1900. 

Run live view and make sure that the background fills the field of view of your camera, then click Background. Now place your specimen in the field of view and click Specimen. You can repeat this process for Background/Speciment many times to improve the detection. Now the background/speciment can be automatically detected. Reset is used to reset the detected images, Show Image(s) opens images used for the Background detection. Buttons Capture and Live can be used for testing the detection.

Automatic background detection can now be used in several tasks, e.g.: Autofocus + Focus Surface > Autofocus Settings. Its label â€œIsBackgroundâ€ is stored in the database and can be used in the job e.g. as a condition.

HCA/JOBS > Acquisition > Parallelize Device Movement

(requires: JOBS Viewer)

This command is used to run devices in parallel (e.g. Stage and Filter Wheel moves at the same time) to speed up the acquisition either during a job run or in ND Acquisition using a Ti Recipe. If a device producing recipe signal is connected (i.e. Ti-E, NIDAQ), the devices start the parallel movement just on â€œexposure endâ€ signal instead of â€œafter captureâ€ in Jobs without Ti Recipe.

HCA/JOBS > Database > Create New Empty

(requires: JOBS Viewer)

Creates a new Jobs database. Specify the location of the database files, name the .dat file and click Save.

HCA/JOBS > Database > Create New Based on Current

(requires: JOBS Viewer)

Opens a dialog window where you can select which items of the current database will be copied to the new database.

HCA/JOBS > Database > Open Existing

(requires: JOBS Viewer)

Shows the Open Database File window enabling to load a Jobs database from a .dat file.

HCA/JOBS > Database > Rename

(requires: JOBS Viewer)

Opens the Database Rename dialog window which can be used to rename the currently opened database.

HCA/JOBS > Database > Quick Open

(requires: JOBS Viewer)

Shows a dialog window enabling to load an existing database from a .dat file.

HCA/JOBS > Database > Refresh

(requires: JOBS Viewer)

This function reloads the currently opened database. If any changes are made to your database remotely (more users working on one database), use this function to promptly reload the database so that it is up to date.

HCA/JOBS > Database > Cleanup

(requires: JOBS Viewer)

Frequent database inserts, updates, and deletes can cause the database file to become fragmented. This function rebuilds the entire database to improve the speed when working with the database.

HCA/JOBS > Database > Backup

(requires: JOBS Viewer)

This command enables saving all files you have created using Jobs into a desired folder, see Database Backup.

HCA/JOBS > Database > Restore

(requires: JOBS Viewer)

This operation replaces the contents of the current database folder with the contents of the backup folder.

HCA/JOBS > Database > Upload

(requires: JOBS Remote Database)

This window opens the Upload dialog. Click ... and specify where the Remote database .dat file is present. Use the check boxes next to each job to include/exclude them into/from the uploading process. The Uploading Rules table below shows the compared data status of both the local and the remote databases. Adjust the Action to perform combo boxes to set the merging rules and then click Upload to start the uploading process.

HCA/JOBS > Database > Download

(requires: JOBS Remote Database)

This window opens the Download dialog. Click ... and specify where the Remote database .dat file is present. Use the check boxes next to each job to include/exclude them into/from the downloading process. The Downloading Rules table below shows the compared data status of both the local and the remote databases. Adjust the Action to perform combo boxes to set the merging rules and then click Download to start the downloading process.

HCA/JOBS > Database > Properties

(requires: JOBS Viewer)

For more information about database properties go to Database Properties.

HCA/JOBS > Database > Upgrade Database

(requires: JOBS Viewer)

If this command is displayed, a new version of NIS-Elements with an updated database structure was installed. Run the command in order to use the database with the new version.

Caution

Once the database is upgraded, it cannot be accessed by older versions of NIS-Elements.
